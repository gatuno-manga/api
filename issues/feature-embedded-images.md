# Issue: Implement Asynchronous Processing for Embedded Images in Text (Markdown/HTML)

## Context

Currently, the Gatuno API strictly processes and mirrors images generated by the scraping of standard chapter
pages. However, when users or systems upload text-based chapters (Web Novels in Markdown/HTML) or insert images
within comments, the application saves the raw strings to the database without analyzing their contents.

This causes the frontend/mobile app to render images directly from external servers (Hotlinking). This
architectural gap makes the platform vulnerable to external dependency, link rot, privacy leaks (tracking
pixels), and retaliation from external hosts (replacing images with malicious content).

## Objective

Implement an asynchronous text-processing pipeline capable of parsing Markdown and HTML contents upon creation.
The pipeline must detect external image URLs, download and optimize the assets into Gatuno's own S3 storage,
and rewrite the original text to point to the new internal URLs.

## Technical Details

### 1. Risk Mitigation

- **Link Rot Prevention:** Ensures Web Novel chapters and visual comments remain intact forever, even if the
  source website goes offline.
- **Privacy & Security:** Prevents third-party servers from collecting IP addresses and User-Agents from
  Gatuno's readers via Tracking Pixels.
- **Offline Support:** Mobile apps can confidently pre-fetch the text and all its internally-hosted image
  dependencies for offline reading.

### 2. Processing Pipeline (Asynchronous)

Since downloading images during an HTTP Request is a blocking and slow operation, the process must be delegated
to background workers:

1.  The user uploads a Markdown/HTML chapter or posts a comment. The API saves the text as-is and returns `200
OK` to the user instantly.
2.  An event is dispatched to the message broker (e.g., `content.text.processing_requested`).
3.  A background worker picks up the job and parses the content (using an AST parser for Markdown or a DOM
    parser for HTML).
4.  The worker extracts all `<img>` `src` attributes and Markdown `![]()` URLs that begin with `http/https` and
    do not belong to the Gatuno domain.

### 3. Image Mirroring and Rewriting

1.  The worker downloads the extracted external images.
2.  It pushes the images to Gatuno's processing microservice (or directly optimizes them and uploads to S3).
3.  The worker performs a string replacement or AST node update on the original text, swapping the external URL
    with the new internal S3 URL.
4.  The worker updates the database record with the new, sanitized text.

## Tasks

- [ ] Research and integrate an AST Markdown parser (e.g., `remark` or `marked`) and an HTML parser (e.g.,
      `cheerio`) compatible with Node.js.
- [ ] Create a BullMQ/Kafka consumer dedicated to text processing.
- [ ] Implement the downloader logic to handle external images found in the text (with strict timeouts and file
      size limits to prevent abuse).
- [ ] Implement the integration with the S3 uploader / Image Processing Go Microservice for these specific
      assets.
- [ ] Create the database update routine to safely overwrite the old text with the mirrored URLs.
- [ ] Ensure that subsequent edits to the comment/chapter trigger the parser again to catch newly added images.
